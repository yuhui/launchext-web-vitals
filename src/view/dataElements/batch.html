You cannot save this Data Element because you have not enabled batch collection of Web Vitals metric reports in the extension configuration.

To enable batch collection of Web Vitals metric reports:

  1. In Extensions, open the "Web Vitals" extension configuration.
  2. Expand the "Advanced" section.
  3. Enable the "Collect batch of Web Vitals metric reports" toggle.
  4. Save the extension configuration.

Then, return here to use this Data Element type.

No configuration necessary

How this Data Element works

This Data Element returns the batch of Web Vitals metric reports that have been collected so far.

How to use:

  • %data element name%
  • _satellite.getVar('data element name')

The batch is an array of objects, where each object contains a Web Vitals metric report with the following information:

Key Data type Value
id String Unique ID representing this reported metric that's specific to the current web page shown in the user's web browser.
name String Name of the reported metric in acronym form.
fullName String Full name of the reported metric.
value Number Current value of the reported metric.
rating String Rating of the reported metric.
delta Number Delta between the current value and the last-reported value of the reported metric.
entries Various Array of the performance entries used in the reported metric's value calculation. (The actual data type depends on the reported metric.)
navigationType String Navigation type of the reported metric.
attribution Object Potentially-helpful debugging information that can be sent along with the metric value for the current page visit in order to help identify issues happening to real-users in the field. Learn more about the attribution metric property.
