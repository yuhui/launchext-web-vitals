How this Event works

This Event is triggered at the first point in the current page's load timeline where the user can see anything on the screen. Learn more about First Contentful Paint (FCP).

In most cases, you will need to report only the delta (the difference between the current value and the last-reported value). You can then compute the total value by summing all metric deltas sent with the same ID in your analytics system. Learn more.

This Event works with the following web browsers only:

  • Chromium-based browsers, e.g. Google Chrome, Microsoft Edge, Opera, Samsung Internet
  • Firefox
  • Safari

You are advised to send your analytics data with a beacon. Learn more.
With Adobe Analytics, you can do this by setting s.useBeacon = true; in your "Set Variables" Action.

Accuracy of measurements Since web-vitals.js is loaded after the Launch library itself has been loaded, some Web Vitals metrics may not be measured as accurately as when they are implemented directly into the web pages. In some cases, the Web Vitals metrics may just not be measured at all.

The following Data Elements are available for use with your Rule's Conditions and Actions.

Data Element Data type Value
Metric ID String Unique ID representing this measured metric that's specific to the current web page shown in the user's web browser
Metric Name String Name of the measured metric in acronym form: FCP.
Metric Full Name String Full name of the measured metric: First Contentful Paint.
Metric Value Number Current value of the measured metric.
Metric Rating String Rating of the measured metric. It can be one of the following:
  • good
  • needs-improvement
  • poor
Metric Delta Number Delta between the current value and the last-reported value of the measured metric.
Metric Entries Array of PerformancePaintTimings Array of the performance entries used in the measured metric's value calculation.
Metric Navigation Type String Navigation type of the measured metric. It can be one of the following:
  • navigate
  • reload
  • back-forward
  • back-forward-cache
  • prerender
Metric Attributions Various Potentially-helpful debugging information that can be sent along with the metric value for the current page visit in order to help identify issues happening to real-users in the field.
Refer to the "Metric Attribution" Data Element type in this extension for more information about the allowed attributions:
  • Time to First Byte
  • First Byte to FCP
  • Load State
  • FCP Entry
  • Navigation Entry
